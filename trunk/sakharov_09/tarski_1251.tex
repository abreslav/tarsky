Как известно, не существует алгоритма, позволяющего проверить истинность произвольного утверждения о натуральных числах \cite{godel}. Неразрешима даже задача о существовании корней произвольного диофантова уравнения \cite{hilbert10}.

Однако для вещественных чисел подобные задачи становятся разрешимыми. В 1948 году американский логик Альфред Тарский опубликовал алгоритм, позволяющий проверить истинность утверждения о полиномах с рациональными коэффициентами и вещественными переменными, построенного с помощью строгих и нестрогих сравнений, логических связок (``и'', ``или'', ``не'' и т.д.), а также кванторов существования и всеобщности \cite{matiyas}.

Вот пример такого утверждения:
\begin{equation}\exists x \left\{ \left(x > 0\right) \wedge \left(x^3 - 3x  > 1\right) \rightarrow \left(x^2 + \frac{1}{32}x^2 + \frac{2}{136} x < 3\right)\right\} \label{f1} \end{equation}

С помощью таких утверждений можно сформулировать (а с помощью алгоритма -- автоматически доказать) многие геометрические теоремы. 

% * Мы реализовали алгоритм для одной переменной на языке Pascal  (Delphi)
% * Программа понимает такой входной язык 
Целью данного проекта являлась реализация алгоритма Тарского для случая одной переменной. Алгоритм реализован на языке Object Pascal (Delphi). Программа получает на вход формулу в текстовой форме. Текстовый вид формулы (\ref{f1}) приведен ниже:
\begin{verbatim}
E x {
  [x > 0
  and
  x^3 - 3 * x  > 1]
  -->
  x ^ 2 + 1/32 * x^2 + 2/136 * x < 3
}
\end{verbatim}
% * выдает ответ и доп. информацию
Результатом работы программы является сообщение об истинности формулы, а также информации о времени работы и затраченных ресурсах. В дополнение к этой информации строятся графики полиномов, указанных в условии.

% * для этих целей мы реализовали 
%  * арифметику длинных целых
%  * арифметику рациональных
%  * арифметику полиномов
%  * построение насыщенной системы
%    * непосредственное с небольшими оптимизациями
%  * индуктивное заполнение таблицы знаков полиномов
%  * вычисление значения формулы по таблице
Представляемая реализация включает такие подзадачи как
\begin{itemize}
	\item арифметика длинных чисел (натуральных и рациональных);
	\item арифметика полиномов;
	\item дополнение данной системы полиномов до насыщенной системы;
	\item вычисление значений формулы по таблице знаков полиномов.
\end{itemize}

% * алгоритм имеет экспоненциальную нижнюю оценку [], мы применяем такие оптимизации
%  * не используем производную произведения, а просто вставляем столбцы между соседними корнями
%  * выбрасываем дубликаты из насыщенной системы
Алгоритм Тарского имеет экспоненциальную нижнюю оценку на время работы \cite{matiyas}, тем не менее на не слишком больших примерах представляемая реализация показывает приемлемое время работы, так пример (\ref{f1}) обрабатывается за несколько миллисекунд.

Быстродействие системы сильно зависит от объема насыщенной системы, поэтому при ее построении используются дополнительные эвристики для исключения дубликатов и сокращения количества делений. 

Описание алгоритма Тарского, предложенное в \cite{matiyas} предполагает добавление к исходной системе полиномов производной их произведения, что значительно повышает сложность задачи, увеличивая размер насыщенной системы. В предлагаемой реализации этого удалось избежать с помощью добавления в таблицу знаков полиномов фиктивных столбцов между корнями исходных многочленов. Такая оптимизация привела к многократному сокращению времени работы. Кроме таких ``алгоритмических'' оптимизаций, применялись также чисто технические оптимизации на низком уровне абстракции.

Представляемая реализация во многих случаях позволяет решать задачи, вовлекающие полиномы достаточно больших степеней, так некоторые полиномы 40 степени удается обработать менее, чем за 6 секунд.
